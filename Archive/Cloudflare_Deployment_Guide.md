# Cloudflare × GitHub 法人サイト構築ガイド

このドキュメントは、実際のチャット履歴（作業ログ）と、Cloudflareの価値に関する資料を統合した、再現可能な手順書です。

## 1. なぜCloudflareなのか？（構築の価値）

従来の開発手法（レンタルサーバーやVercel等）と比較して、以下のメリットがあります。

*   **グローバル展開**: 世界300拠点以上のエッジでコンテンツを配信するため、どこからアクセスしても爆速です。
*   **コスト効率**: 静的サイトホスティング（Pages）は無料枠が非常に寛容です。
*   **スケーラビリティ**: 将来的にAI機能（Workers + Gemini API）やデータベース（KV/D1）を組み込む際も、サーバーレスでシームレスに拡張可能です。

## 2. 構築手順（実録）

初心者でも再現できるように、今回実際に通った手順を整理しました。

### STEP 0: 環境準備

まず、開発に必要なツールをインストールします。Windows環境では `winget` を使うと簡単です。

1.  **ツールのインストール**（PowerShellで実行）
    ```powershell
    winget install Git.Git
    winget install GitHub.cli
    ```
    ※ インストール後、ターミナルの再起動が必要です。

2.  **GitHub認証**
    ```powershell
    gh auth login
    ```
    - `GitHub.com` > `HTTPS` > `Yes` > `Login with a web browser` を選択。
    - 表示されたコード（例: `ACDC-AD22`）をブラウザに入力して認証します。

3.  **Gitユーザー設定**
    ```bash
    git config --global user.name "あなたのユーザー名"
    git config --global user.email "あなたのメールアドレス"
    ```

### STEP 1: プロジェクトの用意

今回は「ハラスメント対策協会」のサンプルサイトを構築しました。

1.  **フォルダ構成**
    ローカルに `site` というフォルダを作成し、以下のファイルを配置します。
    - `index.html`: トップページ
    - `css/style.css`: デザイン設定
    - `js/main.js`: 動作スクリプト

### STEP 2: GitHubリポジトリの作成とアップロード

コマンドライン（ターミナル）から一気にリポジトリ作成とアップロードを行います。

1.  **リポジトリの作成とプッシュ**
    ```bash
    cd site
    git init
    git add .
    git commit -m "Initial commit"
    gh repo create harassment-prevention-org-sample --public --source=. --push
    ```
    ※ `gh` コマンドを使うと、ブラウザを開かずにリポジトリ作成から連携まで完了できます。

### STEP 3: Cloudflare Pages での公開

ここからが重要です。Cloudflareのダッシュボードで設定を行います。

1.  **Cloudflareにログイン**
    [Cloudflare Dashboard](https://dash.cloudflare.com) にアクセス。

2.  **プロジェクトの作成（重要！）**
    - サイドメニューの「Workers & Pages」をクリック。
    - 「Create application」ボタンをクリック。
    - **【重要】ここで「Pages」タブを選択してください。**
        - デフォルトは「Workers」タブになっています。「Deploy command: npx wrangler deploy」と表示されていたら**間違い**です。
        - 必ず「Pages」タブに切り替えてから「Connect to Git」を押してください。

3.  **リポジトリの接続**
    - GitHubアカウントと連携し、作成したリポジトリ（`harassment-prevention-org-sample`）を選択します。

4.  **ビルド設定**
    - 今回のような静的サイト（HTML/CSS）の場合、設定は**デフォルトのままでOK**です。
    - **Build command**: 空欄
    - **Build output directory**: 空欄
    - 「Save and Deploy」をクリック。

5.  **完了**
    - 「Success!」と表示されたら公開完了です。発行されたURL（`https://xxx.pages.dev`）にアクセスして確認します。

## 4. 初心者がハマりやすいポイント（回避策）

今回の作業振り返りから、特に注意すべき点をまとめました。

1.  **「Workers」と「Pages」の間違い** (★最重要)
    *   **現象**: Cloudflareで「Deploy command」を聞かれる、または画面が違う。
    *   **原因**: デフォルトで「Workers」タブが開いているため。
    *   **対策**: 必ず画面上部の **「Pages」タブをクリック** してから「Connect to Git」を押すこと。

2.  **ターミナルの再起動忘れ**
    *   **現象**: `git` や `gh` をインストールしたのに「コマンドが見つかりません」と言われる。
    *   **原因**: インストール情報は、ターミナル（黒い画面）を一度閉じないと反映されないため。
    *   **対策**: ツールを入れたら、**必ずVS Codeやターミナルを一度完全に閉じて、開き直す**。

3.  **フォルダの場所迷子**
    *   **現象**: コマンドを打っても「ファイルがありません」と言われる。
    *   **原因**: VS Codeで開いているフォルダが、作業したいプロジェクトの場所とズレている。
    *   **対策**: `cd` コマンドで移動するか、VS Codeの「ファイル」→「フォルダを開く」で、**プロジェクトのフォルダそのもの（今回は `site`）を直接開く**。

4.  **GitHub認証のコード入力**
    *   **現象**: 「First copy your one-time code」と言われて止まってしまう。
    *   **対策**: 表示された8桁の英数字（例: `ACDC-AD22`）をコピーし、自動で開くブラウザ画面に貼り付ける必要があることを事前に伝えておく。

## 5. トラブルシューティング

実際に遭遇した点への対策です。

*   **Q: `gh` コマンドが使えない**
    *   A: インストール後にターミナル（PowerShell）を再起動していない可能性があります。一度閉じて開き直してください。
*   **Q: Cloudflareでデプロイコマンドを聞かれる**
    *   A: 「Workers」を作成しようとしています。戻って「Pages」タブを選び直してください。

*   **Q: 黒い画面（コマンドライン）での操作は必須ですか？**
    *   A: **必須ではありませんが、推奨されます。**
        *   **Git (Git for Windows)**: これは「保存機能」そのものなので、パソコンで開発するには**ほぼ必須**です。
            *   ※インストーラーに「Git GUI」が含まれていますが、これは自動で入る「おまけ」のようなものです。今回は使いませんでしたが、インストールされていて問題ありません。
        *   **GitHub CLI (gh)**: 今回「認証」や「リポジトリ作成」を簡単にするためにインストールしました。これのおかげで、ブラウザを行き来したり複雑なパスワード設定をせずに済みました。
        *   **もっと楽な方法**: **GitHub Desktop** というアプリを使うと、これらの黒い画面操作をマウスだけで代用できます。次回以降はこちらを検討しても良いでしょう。
